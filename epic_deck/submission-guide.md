# Submission Guide

Use this guide when teammates send deck materials for the Epic keynote.

## Preferred Rule

Send `content`, not finished slide design.

If a page is still rough, that is fine. Clear inputs are more useful than polished but inconsistent layouts.

## What To Send

- source deck pages or PPT/PDF/HTML exports,
- page copy or speaking points,
- data tables and source links,
- screenshots, references, logos, and charts,
- notes on what is must-keep versus optional.

## Where To Put It

- if the correct section is unclear, put it in root `00_inbox/`
- if the section is known, put it in that section's `00_inbox/`

## Naming

Use clear versioned filenames.

Examples:

- `fn_market_gtm_v01.pptx`
- `fn_creator_ecosystem_notes_v02.docx`
- `fn_offer_terms_v01.pdf`

Avoid names like:

- `final.pptx`
- `latest2.pdf`
- `new version.pptx`

## Helpful Notes To Include

- which section this belongs to,
- what the page is trying to prove,
- what absolutely must stay,
- what is still missing or unverified,
- who owns follow-up questions.
