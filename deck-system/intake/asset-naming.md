# Asset Naming Guide

## Purpose

Consistent asset naming makes deck assembly faster and reduces ambiguity during revision.

## Naming Format

Use:

`[project]_[section]_[topic]_[asset-type]_[version]`

Example:

`fortnitechina_market_zerobuild_screenshot_v01`

## Recommended Asset Types

- `screenshot`
- `chart`
- `logo`
- `photo`
- `render`
- `reference`

## Rules

- use lowercase letters,
- use hyphen-free concise words where possible,
- separate parts with underscores,
- include a version suffix,
- avoid generic names like `image1`, `final2`, or `latest`.

## Required Metadata

For any shared asset, also track:

- owner,
- source,
- usage rights or restrictions,
- final or placeholder status.
