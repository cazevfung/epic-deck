# Start Here

Use this file when you want the fastest correct way into the deck system.

## If You Are Starting A New Deck Task

Read these in order:

1. [`docs/00-overview.md`](docs/00-overview.md)
2. [`docs/01-core-workflow.md`](docs/01-core-workflow.md)
3. [`docs/08-visual-system.md`](docs/08-visual-system.md)
4. [`docs/03-page-templates.md`](docs/03-page-templates.md)
5. [`docs/04-page-recipes.md`](docs/04-page-recipes.md)
6. [`docs/11-quality-checklist.md`](docs/11-quality-checklist.md)

These files define:

- how to execute the work,
- what the deck should feel like,
- which page family to use,
- how to avoid overflow and system drift.

## If You Want To Tell Codex What To Do

Use this command:

```text
Use `deck-system/docs/01-core-workflow.md` for this deck.
Follow the core docs `00-04`, `08`, and `11`.
Start with normalization and audit before redesign.
Assign a template, visual mode, and recipe where needed.
Do not redesign the whole deck in one pass.
Do not solve fit with hidden overflow or global shrinkage.
```

## If You Only Want Intake And Audit

Use this command:

```text
Use `deck-system/docs/01-core-workflow.md` for this deck.
Only do phases 1-3.
Create normalized briefs, audit the slides, assign templates, visual modes, and recipes, and call out fit risks.
Do not rebuild slides yet.
```
