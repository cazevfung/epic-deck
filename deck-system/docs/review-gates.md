# Review Gates

## Purpose

Use this after rebuilding any slide batch.

This is a shorter review layer focused on the failure modes we are actually seeing.

## Gate 1: Bible Similarity

- Does the page clearly resemble a real family from `style-bible.html`?
- Can we point to the family without making excuses?
- Is the page borrowing concrete hierarchy and composition, not just colors?

If not, the page drifted.

## Gate 2: Authored Presence

- Is there one dominant visual move?
- Does the page feel presentation-first rather than document-first?
- Is the page still bold enough to feel like a pitch deck?

If everything is evenly weighted, the page has been flattened.

## Gate 3: Anti-Generic Check

- Did the page collapse into a safe split layout?
- Did we overuse cards where the bible used fields, image pressure, or free composition?
- Did we replace asymmetry with symmetry for convenience?

If yes, redesign before polish.

## Gate 4: Source Preservation

- Are all important claims still present?
- Are the main proof assets still present or intentionally replaced?
- Did cleanup remove actual persuasion?

If evidence disappeared, the page is not done.

## Gate 5: Fit

- Does everything sit inside the safe area?
- Does the page fit at `1920 x 1080`?
- Is any meaningful content hidden, clipped, or quietly shrunk away?

If fit only works because hierarchy was weakened, the page is not done.

## Gate 6: Batch Coherence

- Does the page belong to the same batch as nearby rebuilt pages?
- Is the variety intentional rather than random?
- Are family shifts meaningful rather than accidental?

The deck should feel authored across pages, not identical and not chaotic.

## Stop Conditions

Stop and rework the page if:

- it only matches the bible at the color level,
- it is more generic than the source,
- it uses components as decoration,
- it became cleaner by dropping proof,
- it feels like a system demo instead of a real slide.
